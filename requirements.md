## 8. 페이지별 상세 요구사항

### 8.1 홈 페이지 (`/`)

**구성 요소:**
- Hero 섹션: 큰 제목 + 부제목 + CTA 버튼 2개 ("작업 보기", "연락하기")
- 대표 프로젝트 3개 카드형 노출
- 간단한 자기소개 1문단
- 연락처 CTA 배너

**예시 문구:** 
> "25년차 개발자이자 베스트셀러 작가 홍성호입니다."
> "작가의 꿈을 디지털 세계에 구현해 드립니다."

### 8.2 소개 페이지 (`/about`)

**구성 요소:**
- 프로필 사진 + 자기소개
- 경력 타임라인 (회사명, 기간, 역할)
- 작가로서 경력 (책 제목, 출간일, 성과)

### 8.3 연락처 페이지 (`/contact`)

**구성 요소:**
- 문의 폼 (이름, 이메일, 휴대폰 번호(선택), 제목, 메시지)
- 대표 이메일 주소 (kummasa.naver.com)
- SNS 링크 
  블로그 주소 : https://blog.naver.com/kummasa

- 디자인은 화려함보다 **가독성과 신뢰감**을 우선

- 모든 이미지는 **WebP 포맷**으로 최적화
- 외부 라이브러리는 **최소한**으로 사용

## 12. Claude Code 작업 시 규칙

> 이 섹션은 Claude Code가 코드 생성할 때 따를 규칙입니다.

- 모든 코드는 **TypeScript**로 작성
- 컴포넌트는 `components/` 폴더에 분리
- 페이지는 `app/` 폴더 (App Router 구조)
- 스타일은 **Tailwind CSS 클래스**만 사용 (인라인 CSS 금지)
- 한글 변수명/주석은 피하고 **영문**으로 통일
- 커밋 메시지는 **Conventional Commits** 형식 (feat:, fix:, style: 등)
- 새 기능 추가 후 반드시 **로컬 테스트** 안내
